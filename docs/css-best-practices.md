# 網頁排版與 CSS 防坑心法 (CSS Layout Best Practices)

這份文件記錄了在開發 UI 時，如何避免常見的 CSS 跑版、溢出與邊界錯誤，確保元件在各種螢幕尺寸下都能具備高可用性與彈性。

## 1. 「不要關住內容」：偏好內距 (Padding) 而非固定尺寸 (Fixed Size)
當你寫死 `h-11`（固定高度）或 `w-64`（固定寬度）時，你就等於把內容關進了一個死胡同。只要字體變大、翻譯字串變長，或是不同瀏覽器對行高 (line-height) 的渲染有差異，內容就會撞到牆壁甚至溢出。

**✅ 規避心法**：
讓內容撐開容器，而不是讓容器去限制內容。盡量使用 `h-auto` 搭配 `padding` (例如 `px-6 py-3`)。這樣無論內容怎麼變化，元件永遠會保持完美的呼吸空間。除了特定形狀（如正圓形 Avatar 或 Icon）之外，盡量少寫死 `width` 和 `height`。

## 2. 「Flexbox 的隱藏陷阱」：永遠記住 `min-w-0`
當你把元素放進 Flex 容器時，Flex 子元素有一個預設的隱藏行為：**它們寧死也不會縮得比自己的內容還小**（這叫做內建的 `min-width: auto`）。這會導致就算你寫了換行 (`break-words`) 或是截斷 (`truncate`)，遇到超長不中斷字串時，它還是會硬生生地把父元素撐破。

**✅ 規避心法**：
當你在 Flex 排版中遇到子元素撐破父元素時，第一直覺就是去給那個子元素加上 `min-w-0` (或是 `overflow-hidden`)。這會打破它「抵死不從」的脾氣，讓它乖乖聽從外層容器的寬度限制並正確換行。

## 3. 「統一兵權」：不要混用 Inline Style 與 CSS 類別
當你在同一個屬性上混用多種樣式系統時（例如：用 Tailwind 管電腦版 `md:translate-x-0`，卻用 React Inline Style 管手機版 `style={{ transform: ... }}`），Inline Style 的超高權重 (Specificity) 會直接霸凌掉 CSS 類別，導致你的 RWD (響應式設計) 在特定斷點直接失效。

**✅ 規避心法**：
絕對不要在同一個屬性上混用兩種樣式系統。如果非得用動態變數（例如 JS 計算出來的滑動距離），請把它寫成 **CSS 變數** (`style={{ '--mobile-translate': ... }}`)，然後讓 Tailwind 或 CSS 去讀取這個變數。這樣才能保持權重的乾淨與統一。

## 4. 養成「破壞性測試 (Stress Testing)」的習慣
人類開發者通常只看著「完美狀態」的畫面開發，但實際使用場景充滿了極端狀況。

**✅ 規避心法**：
在開發任何 UI 元件完成後，請立刻對它做三件事：
1. **灌爆它**：把標題改成「超級長長長長長長長長長長長長的測試字串」，看它會不會換行或跑版。
2. **擠壓它**：把瀏覽器視窗狂拉到最窄（例如 320px 的 iPhone SE 尺寸），看裡面的按鈕會不會疊在一起。
3. **拔掉它**：把資料清空或是把陣列變成空陣列，看有沒有寫 Empty State（空狀態），還是會直接畫面空白甚至噴出 Undefined 錯誤。
